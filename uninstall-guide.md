If you've read this far, you've already decided that OpenClaw's costs and risks far outweigh its benefits, and you want to say a graceful goodbye to this "lobster." It's not impossible — it's just not as simple as dragging it to the trash.

There are two paths to uninstall: if the CLI is still available, take the easy path; if the CLI is gone but the service is still running, take the manual cleanup path.

**Easy Path (CLI still available)**

The simplest way is to use its built-in uninstall command:

openclaw uninstall

To wipe all configurations in one go and skip all confirmation prompts, add flags:

openclaw uninstall --all --yes --non-interactive

If you prefer npx, you can also run:

npx -y openclaw uninstall --all --yes --non-interactive

If you'd rather do it step by step manually, the result is exactly the same. Execute in order:

Step 1 — Stop the gateway service:

openclaw gateway stop

Step 2 — Uninstall the gateway service itself:

openclaw gateway uninstall

Step 3 — Delete local state and configuration files:

rm -rf "${OPENCLAW_STATE_DIR:-$HOME/.openclaw}"

Note: If you set OPENCLAW_CONFIG_PATH to a custom path outside the state directory, that file also needs to be deleted manually, otherwise it will remain as residual.

Step 4 — Delete the workspace (optional but recommended; this also clears files generated by the Agent at runtime):

rm -rf ~/.openclaw/workspace

Step 5 — Uninstall the CLI itself. Choose the command matching your original installation method:

\# Installed via npm
npm rm -g openclaw
\# Installed via pnpm
pnpm remove -g openclaw
\# Installed via bun
bun remove -g openclaw

If you also installed the macOS desktop app, remove it too:

rm -rf /Applications/OpenClaw.app

**Manual Cleanup Path (CLI no longer available, but service still running)**

If the CLI is gone but the gateway service is still running in the background, you'll need to handle it differently depending on your operating system.

macOS users:

The default service label is _ai.openclaw.gateway_. Execute:

launchctl bootout gui/$UID/ai.openclaw.gatewayrm -f ~/Library/LaunchAgents/ai.openclaw.gateway.plist

If you used the --profile flag, replace the label and plist filename in the command with ai.openclaw.<profile-name>. Also, if any legacy com.openclaw.* format plist files from older OpenClaw versions still exist, delete those too.

Linux users:

The default service unit name is _openclaw-gateway.service_. Execute:

systemctl --user disable --now openclaw-gateway.service
rm -f ~/.config/systemd/user/openclaw-gateway.service
systemctl --user daemon-reload

If you used --profile, the corresponding unit name is openclaw-gateway-<profile-name>.service — just substitute it in the commands.

Windows users:

The default task name is OpenClaw Gateway. Execute:

schtasks /Delete /F /TN "OpenClaw Gateway"Remove-Item -Force "$env:USERPROFILE\.openclaw\gateway.cmd"

If you used --profile, the task name is OpenClaw Gateway (<profile-name>), and also delete ~\.openclaw-<profile-name>\gateway.cmd.

A few details that are easy to miss



Multiple profiles: If you created multiple configurations using the --profile flag, each profile has its own independent state directory. The default path is _~/.openclaw-_<profile-name>. You need to find and delete each one individually. Miss even one, and residual data remains.



Remote mode: If you were using remote mode, the state directory isn't on your local machine — it's on the gateway host. This means the steps for stopping the service and deleting the state directory need to be executed on the gateway host. Local operations alone won't be enough.



Source installation: If you originally ran OpenClaw via git clone from source, the uninstall order matters — you must uninstall the gateway service first (via the easy path or manual cleanup path above), then delete the repository directory, and finally clean up state and workspace. Reversing the order means the service is still hanging around, and deleting the repo actually makes cleanup harder.

Once all of this is done, you've truly said goodbye to the lobster.
